# MaterialYou.Avalonia — Domain Glossary

## Project Identity

| Term | Definition |
|------|-----------|
| **MaterialYou.Avalonia** | An Avalonia UI theme library implementing Material Design 3 (Material You) design system. |
| **Genre** | Mixed — retains Semi.Avalonia's project skeleton and build infrastructure but replaces all theme content with Material 3 implementations. |
| **License** | Apache-2.0 |

## Architecture

| Term | Definition |
|------|-----------|
| **DynamicColor subsystem** | An internal C# project (`MaterialYou.Avalonia.DynamicColor`) housing the Avalonia integration layer (markup extensions, attached properties) for dynamic color. Underlying HCT/Scheme algorithms come from the `MaterialColorUtilities` NuGet package. Apache-2.0 code adapted from `Shirasagi0012/MaterialColorUtilities`. |
| **MaterialSymbols project** | A separate project (`MaterialYou.Avalonia.MaterialSymbols`) that packages Material Symbols icon paths as XAML PathGeometry resources. Generated by automation scripts from Google's Material Symbols font/design-tokens. Published as its own NuGet package for community reuse. |
| **Controls Direct Binding** | Control templates reference Material 3 design tokens directly via markup extensions (e.g. `{MdSysColor Primary}`). No intermediate semantic resource alias layer (unlike Semi's 3-layer model with Themes/Shared/ + Themes/Light/ + Themes/Dark/). |
| **Design Layers** | Each Material 3 design subsystem has its own directory: Controls/, Tokens/, Shape/, Elevation/, Typography/, Motion/, StateLayers/. |
| **Theme entry** | `MaterialYouTheme` (extends `Styles`) is a pure XAML control template collection with no built-in Scheme. Users set `MaterialColor.Scheme` in code-behind for dynamic color. |

## Color System

| Term | Definition |
|------|-----------|
| **HCT** | Hue/Chroma/Tone — the perceptually accurate color model used by Material 3. Replaces HSV/HSL. |
| **TonalPalette** | A 13-step (0–100) per-hue color ramp derived from a seed color via the HCT model. |
| **Scheme** | A set of 5 roles (Primary, Secondary, Tertiary, Neutral, NeutralVariant) each with tonal palette assignments for on/on-container/container variants, plus Error. Generated at runtime from a seed color. |
| **Seed Color** | The source color from which all scheme colors are algorithmically derived. |
| **MdSysColor** | Markup extension that resolves to a Material 3 system color token bound to the current Scheme and theme variant. |
| **MdRefPalette** | Markup extension that resolves to a specific tone of a reference palette (e.g. Primary at tone 60). |

## Theme Variants

| Term | Definition |
|------|-----------|
| **Light** | Standard light variant — white/light backgrounds, dark text. |
| **Dark** | Standard dark variant — dark backgrounds, light text. |
| **HighContrast** | Increased contrast variant (planned). Uses same scheme but with elevated contrast ratios. |

## Design Subsystems

| Term | Definition |
|------|-----------|
| **Shape** | Material 3 shape family system — 5 levels (None/Small/Medium/Large/Full) each defined as `CornerRadius` resources in `Tokens/Shape.axaml`. Controls reference by level name. |
| **Elevation** | Material 3 elevation system — 6 levels (0–5). Each level maps to a `BoxShadow` value via `{mcu:MdElevation Level}` markup extension. The extension is theme-variant-aware (shadows fade in Dark). Surface tint is pre-blended into surface colors via `{mcu:MdSurface Level}`. Single-element visual tree. |
| **Typography** | Material 3 type scale — predefined `Style` resources per level (Display/Headline/Title/Body/Label × Large/Medium/Small) that bundle FontSize, FontWeight, LineHeight, LetterSpacing. Controls reference via `Style="{StaticResource LabelLarge}"`. |
| **Motion** | Material 3 motion curves + durations — exposed as XAML easing resources and duration constants in `Tokens/Motion.axaml`. Precisely defined (Emphasized/Standard/Linear curves with corresponding durations). |
| **State Layers** | Material 3 interaction state feedback — overlay `Border` embedded in each ControlTemplate at root level. Opacity controlled via pseudo-class selectors (hover: 8%, focus: 12%, pressed: 12%, drag: 16%). |
